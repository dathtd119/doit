# Runbooks

Incident and operational playbooks for **do** (local fork of Grok Build).

| Runbook | When to use |
|---------|-------------|
| [Incident response](./incident-response.md) | Production or user-facing failures in the pager/agent runtime |
| [Release / deploy impact](./deploy-impact.md) | After tagging a release or shipping a harness change |
| [Alert routing](./alerting.md) | How alerts page on-call / open issues |

These runbooks are local documentation pointers so agents and humans share one incident path. Wire them to your Sentry / Mixpanel / pager projects for your environment.
